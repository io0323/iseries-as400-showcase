# システム設計書 — インテリア専門商社 基幹システム

## 1. システム概要

### 1.1 目的
インテリア専門商社の業務効率化を目的とした基幹システム。  
受発注・物流在庫・会計の3業務領域を統合管理する。

### 1.2 システム構成

```
IBM i (AS/400)
├── 受発注管理 (ORDMGR)
│   ├── ORDPF  — 受注物理ファイル
│   ├── ORDLF  — 受注論理ファイル（得意先順）
│   └── ORDMGR.RPGLE — 受注管理プログラム
│
├── 物流在庫管理 (INVMGR)
│   ├── ITMPF  — 商品マスタ物理ファイル
│   ├── INVPF  — 在庫物理ファイル（商品+ロケーション）
│   ├── INVLF  — 在庫論理ファイル（ロケーション順）
│   └── INVMGR.RPGLE — 在庫管理プログラム
│
├── 会計管理 (ACCMGR)
│   ├── ARPF   — 売掛金物理ファイル
│   ├── APPF   — 買掛金物理ファイル
│   └── ACCMGR.RPGLE — 会計管理プログラム
│
└── 共通 (SRVPGM)
    ├── UTILSRV.RPGLE — 共通ユーティリティ
    └── ERRSRV.RPGLE  — エラーハンドリング
```

## 3. プロセス設計

### 3.1 受注〜出荷フロー

```
受注入力（ORDMGR）
    ↓ ORDSTS=1（受注）
在庫引当確認（INVMGR.AllocateStock）
    ↓ 引当成功 → ORDSTS=2, INVPF.ALCQTY+=QTY
出荷処理（INVMGR.Ship）
    ↓ ORDSTS=3, INVPF.STKQTY-=QTY
売掛金生成（ACCMGR.CreateArRecord）
    ↓ ARPF新規レコード作成（消費税10%加算）
入金消込（ACCMGR.Receive）
    ↓ ARPF.RCVAMT+=入金額, BALAMT更新
完済（ARSTS=3）
```

### 3.2 仕入〜支払フロー

```
発注（ORDMGR）
    ↓
仕入登録（ACCMGR）→ APPF新規レコード作成
    ↓
入庫処理（INVMGR.Receive）→ INVPF.STKQTY+=数量
    ↓
支払処理（ACCMGR.Pay）
    ↓ APPF.PAYAMT+=支払額, BALAMT更新
完済（APSTS=3）
```

### 3.3 エラーハンドリング方針

- ファイル操作後は必ず `%Found / %Error / %Eof` で状態確認
- エラーメッセージはERRSRVサービスプログラムで一元管理
- 在庫不足・重複キー等は業務エラーとして画面にメッセージ表示
- システムエラーはジョブログに書込み（SrWriteJobLog）

## 5. セキュリティ設計

- オブジェクト権限：ライブラリレベルで業務担当者グループに付与
- ジョブ実行ユーザー：INTERIORUSER（業務専用ユーザーID）
- パスワードポリシー：IBM i 標準セキュリティレベル40適用
- 監査ログ：UPDDTとUPDUIDフィールドで更新者・日時を全レコードに記録

## 6. パフォーマンス設計

- アクセスパスの最適化：ORDLF（得意先順）・INVLF（ロケーション順）で逐次読込の効率化
- ジョブキューによるバッチ分離：夜間バッチ（月次請求締め）はJOBQで別管理
- RPGのActGrp設定：*NEWでメモリリーク防止、Named ActGrpで再利用効率化
